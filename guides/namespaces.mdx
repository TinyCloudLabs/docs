A namespace is a data ownership container in TinyCloud. Each user has their own namespace where their data is stored and controlled.

## Namespace ID Format

Namespace IDs follow the format:

```
{address}-{chainId}-{prefix}
```

| Component | Description | Example |
|-----------|-------------|---------|
| `address` | Ethereum address (lowercase) | `0x1234...abcd` |
| `chainId` | EIP-155 chain ID | `1` (mainnet), `11155111` (sepolia) |
| `prefix` | Application identifier | `my-app` |

Example namespace ID:
```
0x1234567890abcdef1234567890abcdef12345678-1-my-app
```

## Auto-Creation vs Manual

By default, namespaces are created automatically on first sign-in.

<CodeGroup>
```typescript Web SDK
const tc = new TinyCloudWeb({
  host: 'https://api.tinycloud.xyz',
  prefix: 'my-app',
  // autoCreateNamespace: true (default)
});

await tc.signIn();
// Namespace created automatically if it doesn't exist
```

```typescript Node SDK
const tc = new TinyCloudNode({
  host: 'https://api.tinycloud.xyz',
  prefix: 'my-app',
  privateKey: process.env.WALLET_PRIVATE_KEY,
  autoCreateNamespace: true, // default
});

await tc.signIn();
```
</CodeGroup>

To disable auto-creation:

<CodeGroup>
```typescript Web SDK
const tc = new TinyCloudWeb({
  host: 'https://api.tinycloud.xyz',
  prefix: 'my-app',
  autoCreateNamespace: false,
});

await tc.signIn();
// Throws error if namespace doesn't exist
```

```typescript Node SDK
const tc = new TinyCloudNode({
  host: 'https://api.tinycloud.xyz',
  prefix: 'my-app',
  privateKey: process.env.WALLET_PRIVATE_KEY,
  autoCreateNamespace: false,
});
```
</CodeGroup>

## Getting Namespace ID

After sign-in, retrieve the namespace ID for the current session.

<CodeGroup>
```typescript Web SDK
await tc.signIn();

const namespaceId = tc.userAuthorization.getNamespaceId();
console.log('Namespace:', namespaceId);
// "0x1234...abcd-1-my-app"
```

```typescript Node SDK
await tc.signIn();

const namespaceId = tc.namespaceId;
console.log('Namespace:', namespaceId);
```
</CodeGroup>

## Prefix Usage

The prefix isolates data between different applications for the same user.

```typescript
// User's game data
const gameClient = new TinyCloudWeb({
  host: 'https://api.tinycloud.xyz',
  prefix: 'my-game',
});

// User's notes data (separate namespace)
const notesClient = new TinyCloudWeb({
  host: 'https://api.tinycloud.xyz',
  prefix: 'my-notes',
});
```

Same wallet, different namespaces:
- `0x1234...abcd-1-my-game`
- `0x1234...abcd-1-my-notes`

<Note>
Data in one namespace is completely isolated from other namespaces, even for the same user.
</Note>
