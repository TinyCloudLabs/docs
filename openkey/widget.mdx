Use the SDK-managed widget when a browser app needs to connect a key and ask
the user to sign without building its own OpenKey UI.

## Install

```bash
npm install @openkey/sdk@0.9.0 ethers@6.17.0
```

## Connect an EIP-1193 provider

```typescript
import { OpenKey, OpenKeyProvider } from "@openkey/sdk";
import { BrowserProvider } from "ethers";

const openkey = new OpenKey({
  host: "https://openkey.so",
  appName: "My App",
});
const auth = await openkey.connect();

const eip1193Provider = new OpenKeyProvider(openkey, auth);
const provider = new BrowserProvider(eip1193Provider);
const signer = await provider.getSigner();

const signature = await signer.signMessage("Hello from My App");
console.log(auth.address, signature);
```

`OpenKeyProvider` implements the EIP-1193 methods needed for account discovery,
chain identification, personal message signing, and EIP-712 typed-data
signing. Unsupported provider methods reject with an error.

## Available SDK flows

- `connect()` selects or creates a key
- `signMessage()` signs a personal message
- `signTypedData()` signs EIP-712 typed data
- `linkWallet()` links an external wallet

Managed-key signing is handled by OpenKey. External-key signing is routed to
the wallet provider configured or discovered by the SDK.

## Choose the UI mode

```typescript
const openkey = new OpenKey({
  host: "https://openkey.so",
  appName: "My App",
  mode: "popup",
});
```

Modes are `iframe`, `popup`, and `redirect`. The default iframe mode falls
back to a popup if it does not receive the widget-ready message, which commonly
happens when the app's content security policy blocks the frame.

## Direct widget messaging

The widget routes and `postMessage` message shapes are SDK implementation
details, not a supported public integration contract. Use `@openkey/sdk` so
origin checks, timeouts, popup handling, and message changes remain inside the
SDK. If the SDK cannot support your host environment, describe that constraint
to [TinyCloud support](https://t.me/+-_2ufE0JgIcyMjYx) before depending on the
internal protocol.
