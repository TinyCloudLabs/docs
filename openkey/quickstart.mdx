Use this tutorial to add a working OpenKey connect-and-sign flow to a small
browser app.

## Prerequisites

- Node.js 18 or later
- A browser with WebAuthn support
- A passkey-capable device or hardware security key

## Create the app

```bash
npm create vite@latest openkey-demo -- --template vanilla-ts
cd openkey-demo
npm install
npm install @openkey/sdk@0.9.0
```

Replace `src/main.ts` with:

```typescript
import { OpenKey, type AuthResult } from "@openkey/sdk";

document.querySelector<HTMLDivElement>("#app")!.innerHTML = `
  <button id="connect">Connect OpenKey</button>
  <button id="sign" disabled>Sign message</button>
  <pre id="result">Not connected</pre>
`;

const openkey = new OpenKey({
  host: "https://openkey.so",
  appName: "OpenKey demo",
});

const connectButton = document.querySelector<HTMLButtonElement>("#connect")!;
const signButton = document.querySelector<HTMLButtonElement>("#sign")!;
const result = document.querySelector<HTMLPreElement>("#result")!;
let auth: AuthResult | undefined;

connectButton.addEventListener("click", async () => {
  auth = await openkey.connect();
  result.textContent = `Connected: ${auth.address}`;
  signButton.disabled = false;
});

signButton.addEventListener("click", async () => {
  if (!auth) return;
  const signed = await openkey.signMessage({
    message: "Hello from OpenKey",
    keyId: auth.keyId,
  });
  result.textContent = `Address: ${signed.address}\nSignature: ${signed.signature}`;
});
```

Start the development server:

```bash
npm run dev
```

Open the local URL, select **Connect OpenKey**, and complete the OpenKey flow.
The page should show an Ethereum address. Select **Sign message** and it should
replace that result with the address and a hex-encoded signature.

## What happened

`connect()` returns the selected key's `address`, `keyId`, and `keyType`.
`signMessage()` uses that `keyId` and returns the signature plus the signing
address. The default `iframe` mode falls back to a popup if the app's content
security policy blocks the frame.
